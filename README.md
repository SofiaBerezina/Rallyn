## О проекте

Rallyn — это веб-приложение для публикации, поиска и управления активностями. Сервис помогает пользователям быстро находить интересные события рядом с собой, откликаться на них, вести профиль и управлять собственными объявлениями.

## Какую задачу решает приложение

Приложение упрощает поиск компании и организацию совместных занятий: спорта, прогулок, обучения, досуга и других активностей. Вместо поиска в разрозненных чатах и сообществах пользователь получает единое пространство, где можно публиковать активности, фильтровать их по параметрам и откликаться на подходящие варианты.

## Целевая аудитория

Приложение будет полезно:
- людям, которые хотят найти компанию для совместных занятий;
- тем, кто планирует собственную активность и ищет участников;
- пользователям, которым важно быстро находить события по категории, дате и месту;
- участникам локальных сообществ, спортивных и досуговых групп.

## Краткое описание возможностей

- регистрация и авторизация пользователя;
- просмотр ленты активностей;
- фильтрация активностей по категории, месту и дате;
- создание собственной активности;
- отклики на активности других пользователей;
- просмотр и обработка откликов;
- редактирование профиля пользователя;
- просмотр собственного профиля и профилей других пользователей;
- отображение количества откликов на активность;
- уведомления о ключевых действиях в приложении.

## Используемые технологии и архитектурные подходы

### React + TypeScript

Основной интерфейс приложения построен на **React** и **TypeScript**. Такой стек выбран, потому что он хорошо подходит для модульной фронтенд-архитектуры, позволяет делить интерфейс на переиспользуемые компоненты и снижает количество ошибок за счёт статической типизации.

### Vite

Для сборки и локальной разработки используется **Vite**. Он обеспечивает быстрый старт проекта, быструю пересборку при изменениях и удобен для небольших и средних SPA-приложений.

### TanStack Query

**TanStack Query** используется для работы с серверным состоянием: загрузки, кэширования, обновления и повторного запроса данных. Этот инструмент был выбран вместо ручного управления `useEffect`/`useState`, потому что он:
- упрощает работу с асинхронными запросами;
- уменьшает количество дублирующегося кода;
- автоматически управляет состояниями загрузки и ошибок;
- помогает синхронизировать данные после создания, редактирования и удаления сущностей.

### Context API

**React Context** используется для глобального UI-состояния, которое нужно нескольким частям приложения сразу. В проекте через контекст управляются:
- состояние модальных окон;
- фильтры ленты активностей;
- данные авторизации пользователя.

Такой подход позволяет не прокидывать одни и те же пропсы через множество уровней компонентов и делает архитектуру чище.

### React Router

**React Router** отвечает за маршрутизацию между страницами приложения: вход, лента активностей, профиль и страница 404. Также он помогает разделить публичную и защищённую части приложения.

### Firebase

**Firebase** используется как backend-as-a-service: для регистрации, авторизации и хранения данных пользователей, активностей и откликов. Этот вариант удобен для курсового проекта, потому что не требует отдельного backend-сервера и позволяет быстро собрать рабочее приложение.

### Дополнительные решения

- **sonner** — для красивых уведомлений о действиях пользователя.
- **Яндекс.Метрика** — для аналитики посещаемости и событий.
- **SCSS / Tailwind** — для стилизации и ускорения разработки интерфейса.
- **Локальный state + Context + Query** — комбинированный подход, который помогает отделить UI-состояние от данных, пришедших с сервера.

### API аватаров

В проекте для генерации аватаров используется **DiceBear Avatars API**. Этот сервис был выбран, потому что он позволяет быстро получать аккуратные аватары без отдельной загрузки изображений и хранения файлов на сервере.

Как это реализовано:
- при регистрации пользователя в Firebase в профиль автоматически сохраняется поле `avatar`;
- аватар формируется по ссылке вида `https://api.dicebear.com/.../initials/svg?seed=...`;
- в качестве `seed` используется имя пользователя, поэтому аватар остаётся стабильным и легко узнаваемым;
- если у пользователя нет сохранённого аватара, приложение подставляет DiceBear-ссылку как резервный вариант.

Где используется генерация аватаров:
- при создании нового профиля после регистрации;
- при обновлении имени пользователя, если аватар не был задан вручную;
- в карточке профиля;
- в карточках активностей и откликов, где показывается автор или участник.

Для курсового проекта такой подход удобен, потому что он упрощает MVP: не требуется отдельная логика загрузки файлов, а интерфейс остаётся визуально цельным.

## Основные страницы приложения

- **Страница входа** — авторизация и регистрация.
- **Лента активностей** — список доступных активностей с фильтрами.
- **Профиль пользователя** — информация о пользователе, его активности и редактирование профиля.
- **Страница 404** — отображается при переходе по несуществующему адресу.

## Инструкции по запуску проекта

Проект расположен в папке `activity-platform`.

```bash
cd activity-platform
npm ci
```

### Запуск в режиме разработки

```bash
npm run dev
```

### Сборка проекта

```bash
npm run build
```

### Проверка качества кода

```bash
npm run lint
npm run format:check
npm run test
```

## Тестовые данные для использования приложения

1. Войти с тестовым пользователем или зарегистрировать новый аккаунт на странице входа.
2. После регистрации профиль создаётся автоматически со значениями по умолчанию:
   - имя: `User`
   - описание: `Hello, I am User!`
   - аватар формируется автоматически.
3. Создать несколько тестовых активностей и откликнуться на них.
